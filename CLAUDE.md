# OAM-FanControl

OpenKNX application module for controlling decentralized ventilation fans (e.g. Maico PP B30, Fawas AirSolitaire 160) via KNX bus on RP2040.

## Project Structure

- `src/main.cpp` — Entry point, registers FanModule with OpenKNX framework, dual-core hooks
- `src/Fan.xml` — OpenKNXproducer main config (modules, KO offsets, ETS product definition)
- `src/Fan.conf.xml` — Version config (BASE_VerifyVersion, ReleaseApplicationVersion)
- `include/hardware.h` — Board selector (dispatches to board-specific headers via `DEVICE_*` defines)
- `include/MrSpiebFanControlHardware.h` — Pin assignments for original MrSpieb HW-FanControl board
- `include/Reg1_FanAddon_X2.h` — Pin assignments for OpenKNX Reg1 Fan-Addon-X2 board
- `include/knxprod.h` — Auto-generated by OpenKNXproducer (DO NOT edit manually)
- `lib/OFM-FanControl/` — Fan control module (FanModule, FanChannel, Fan, MaicoPPB30, TachoReader)
- `lib/OGM-Common/` — OpenKNX common framework (v1.7+)
- `lib/OFM-ConfigTransfer/` — Required by OGM-Common v1.7+ for %ProducerModule% expansion
- `restore/` — Dependency restore scripts (clones repos, creates symlinks in lib/)
- `dependencies.txt` — Pinned dependency versions for restore scripts

## Build & Toolchain

- **Platform**: RP2040 (arduino-pico framework) via PlatformIO
- **OpenKNXproducer**: Generates `knxprod.h` and `.knxprod` files from XML configs
  - Binary at `%USERPROFILE%/bin/OpenKNXproducer.exe`
  - CLI: `OpenKNXproducer.exe create -h include/knxprod.h src/Fan.xml`
  - Run via VSCode task "OpenKNXproducer Dev"
  - **Never edit knxprod.h manually** — always regenerate from XML
- **Build**: VSCode task "Build" runs `lib/OGM-Common/scripts/build/OpenKNX-Build.ps1 develop_RP2040`

## Key Conventions

### OpenKNX Module System
- Each module needs a unique `ModuleType` in Fan.xml (BASE=10, UCT=99, FAN=20)
- KoOffset must be >= 20 (0-19 reserved for BASE/Common KOs)
- ETS IDs must NEVER change within an update path (breaks user ETS configurations)
- ConfigTransfer (UCT) module is mandatory for OGM-Common v1.7+ (%ProducerModule% placeholders)

### Dependency Management
- `dependencies.txt` pins commit hashes; `restore/Restore-Dependencies.ps1` clones and symlinks
- Restore script creates detached HEAD — check out named branches before running Build-Dependencies
- Forks: `origin` = mrspieb (upstream), `myfork` = cad435

### Hardware Boards
Two supported boards, selected via `-D DEVICE_*` in platformio.custom.ini:

**MrSpieb HW-FanControl** (default, no DEVICE_ define):
- 2 fan channels, each Maico PPB30 (push-pull with 2 PWM pins + 1 switch pin)
- PWM via SN74 buffer for 3.3V->5V level shifting
- No tachometer inputs

**OpenKNX Reg1 Fan-Addon-X2** (`-D DEVICE_REG1_FAN_ADDON_X2`):
- 2 fan channels, each with single PWM + power switch + tachometer
- S2 PWM pins set to -1 (no push-pull Maico operation)
- Tacho inputs via optocoupler (active low, FALLING edge, 2 pulses/rev)
- Can drive either 1 Maico PPB30 (using both PWM channels) or 2 single-motor fans (e.g. Fawas)
- Requires `-D OPENKNX_DUALCORE` for tacho RPM measurement on core 1

### Dual-Core (RP2040)
- Enabled via `-D OPENKNX_DUALCORE` build flag
- Core 0: KNX stack, fan control logic, KO updates
- Core 1: Tacho ISR pulse counting + RPM calculation (TachoReader)
- `setup1()` waits for core 0 `_setupComplete` before initializing tacho interrupts
- RPM values written to KOs every 1s from core 0 loop

## Communication Objects (KOs)

### BASE KOs (0–7, from OGM-Common)

| KO | Name | DPT | R/W | Description |
|----|------|-----|-----|-------------|
| 1 | Heartbeat | 1.002 | Out | "In Betrieb" — periodic heartbeat telegram |
| 2 | Time | 10.001 | In | Uhrzeit empfangen |
| 3 | Date | 11.001 | In | Datum empfangen |
| 4 | DateTime | 19.001 | In | Kombiniertes Datum/Uhrzeit |
| 5 | IsSummertime | 1.002 | Out | Sommerzeit aktiv |
| 6 | ManualSave | 1.002 | In | Manuelles Speichern auslösen |
| 7 | Diagnose | 16.001 | Out | Diagnose-String |

### FAN KOs (pro Kanal, Channel 1: KO 20–35, Channel 2: KO 36–51)

| Offset | Name | DPT | R/W | Description |
|--------|------|-----|-----|-------------|
| +0 | HumidityInside | 9.007 (%) | In | Relative Luftfeuchte innen. Wird für Automatik-Schwellwertvergleich genutzt. Bei Absolut-Modus zusammen mit Temperatur zur Taupunktberechnung. |
| +1 | TemperatureInside | 9.001 (°C) | In | Temperatur innen. Nur bei Feuchtemessung "Absolut" aktiv — für Taupunktberechnung (absolute Feuchte innen vs. außen). |
| +2 | HumidityOutside | 9.007 (%) | In | Relative Luftfeuchte außen. Nur bei Feuchtemessung "Absolut" — Lüfter läuft nur wenn Taupunkt außen < innen. |
| +3 | TemperatureOutside | 9.001 (°C) | In | Temperatur außen. Nur bei Feuchtemessung "Absolut" — für Taupunktberechnung außen. |
| +4 | Level | 5.010 (0–5) | In | Lüfterstufe direkt setzen (0=aus, 1–5). Setzt Manual Override — blockiert Automatik bis Schwellwert-Kreuzung oder Moduswechsel. |
| +5 | LevelUpDown | 1.007 (Step) | In | Stufe +1 (=1) oder -1 (=0). Gleicher Manual Override wie Level. |
| +6 | LevelFeedback | 5.010 (0–5) | Out | Rückmeldung der aktuellen Lüfterstufe. Wird bei jeder Geschwindigkeitsänderung gesendet (auch automatisch). |
| +7 | OpMode | 1.003 (Enable) | In | Betriebsmodus umschalten: 0=Manuell, 1=Automatik. Nur aktiv wenn ETS-Parameter Betriebsmodus="Kommunikationsobjekt". |
| +8 | OpModeFeedback | 1.003 (Enable) | Out | Rückmeldung Betriebsmodus (0=Manuell, 1=Automatik). |
| +9 | VentMode | 5.010 (0–2) | In | Lüftungsmodus manuell: 0=Wärmerückgewinnung, 1=Zuluft, 2=Abluft. Nur aktiv wenn ETS-Parameter="Kommunikationsobjekt". |
| +10 | VentModeFeedback | 5.010 (0–2) | Out | Rückmeldung Lüftungsmodus (0=WRG, 1=Zuluft, 2=Abluft). |
| +11 | TimerActivation | 1.010 (Start) | In | Timer starten (=1) oder stoppen (=0). Bei Start: Lüfter läuft für konfigurierte Laufzeit, dann automatischer Stopp. Bei Stop: Lüfter sofort aus. |
| +12 | TimerFeedback | 1.011 (State) | Out | Timer-Status: 1=läuft, 0=abgelaufen/gestoppt. Wird automatisch auf 0 gesetzt wenn Timer abläuft. |
| +13 | VentModeAutomatic | 5.010 (0–2) | In | Lüftungsmodus im Automatikbetrieb: 0=WRG, 1=Zuluft, 2=Abluft. Separat vom manuellen Modus konfigurierbar. Nur aktiv wenn ETS-Parameter="Kommunikationsobjekt". |
| +14 | VentModeFeedbackAutomatic | 5.010 (0–2) | Out | Rückmeldung Lüftungsmodus Automatik. |
| +15 | TachoRPM | 7.001 (pulses) | Out | Drehzahl Rückmeldung in RPM. Nur auf Hardware mit Tacho-Eingang (Reg1 Fan-Addon). Wird jede Sekunde aktualisiert, per Bus-Read abfragbar. |

### KO-Sichtbarkeit in ETS (abhängig von Parametern)

Die KOs werden in der ETS dynamisch ein-/ausgeblendet:
- **Betriebsmodus = Aus**: Keine KOs sichtbar
- **Betriebsmodus != Aus**: Level, LevelUpDown, LevelFeedback, Timer immer sichtbar
- **Betriebsmodus = KO**: Zusätzlich OpMode + OpModeFeedback
- **Automatik aktiv** (Modus >= 2): HumidityInside + Automatik-Parameter sichtbar
- **Feuchtemessung = Absolut**: Zusätzlich TemperatureInside, HumidityOutside, TemperatureOutside
- **Lüftungsmodus = KO**: VentMode + Feedback sichtbar
- **Lüftungsmodus Automatik = KO**: VentModeAutomatic + Feedback sichtbar
